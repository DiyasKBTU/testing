# 🌾 AgroFusion Score

**AgroFusion Score** — это прототип системы, которая помогает **оценивать и ранжировать заявки на агросубсидии** не по принципу «кто раньше подал», а опираясь на качество и риски самой заявки.

🏆 **Проект разработан для Decentrathon 5.0** **Трек:** AI for Government | **Кейс:** 2

---

## 🎯 Зачем нужен проект

В текущих реалиях субсидии часто распределяются по очереди подачи. Из-за этого финансирование могут получить не самые эффективные хозяйства, а те, кто просто быстрее отправил документы.

**AgroFusion Score** помогает комиссии:
- 📊 быстро увидеть **рейтинг заявок**;
- 🔍 понять, **почему заявка получила такой балл** (explainability);
- 🌟 выделить **лучших кандидатов**;
- ⚠️ отдельно отметить **рискованные заявки**, требующие ручной проверки.

> **Важно:** Система **не принимает решение вместо человека**, а выступает в роли интеллектуального помощника для комиссии.

---

## ✨ Что умеет система

Прототип уже поддерживает следующий функционал:

- Загрузка заявок на субсидии.
- Расчет итогового балла для каждой заявки.
- Ранжирование заявок от сильных к слабым.
- Формирование **shortlist** кандидатов на одобрение.
- Объяснение результата (explainability) по каждой заявке.
- Выделение потенциально рискованных случаев.
- Загрузка **новых данных** с отдельным анализом.
- Экспорт результатов оценки в CSV-формат.

---

## 🧮 Как считается оценка

Итоговая оценка формируется из нескольких независимых модулей:

1. **История:** Анализ доступных исторических сигналов по заявителю.
2. **Надёжность и соответствие:** Проверка признаков нарушений, рисков и прошлых проблем.
3. **Эко/гео риск:** Оценка рисков, связанных с конкретным регионом, кормовой базой и экологической нагрузкой.
4. **ML-слой:** Дополнительная модель машинного обучения, помогающая лучше различать похожие заявки и повышать точность ранжирования.

## 📝 Что получает пользователь (Вывод системы)

Для каждой заявки в интерфейсе отображается:

- `ID заявки`, `Регион` и `Район`, `Направление субсидии`.
- **Итоговый балл** и **Уровень риска**.
- **Факторы**, которые сильнее всего повлияли на итоговый результат.
- **Рекомендация системы:**
  - 🟢 *Одобрить приоритетно*
  - 🟡 *Одобрить с оговоркой*
  - 🟠 *Нужна ручная проверка*
  - 🔴 *Отложить из-за высокого риска*
  - ⛔ *Отклонить*

## 🚀 Как запустить проект локально

### 1. Создать виртуальное окружение
```bash
python -m venv .venv
```

### 2. Активировать виртуальное окружение
**Для Windows:**
```cmd
.venv\Scripts\activate
```

**Для Linux / macOS:**
```bash
source .venv/bin/activate
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Запустить приложение
```bash
streamlit run agrofusion_score/main.py
```

После запуска интерфейс автоматически откроется в браузере по адресу: `http://localhost:8501`

## 📊 Какие данные используются

**Основной вход:**
* Файл с заявками на субсидии (форматы: `.csv`, `.xlsx`, `.xls`).

**Дополнительные данные (опционально):**
* Данные по поголовью.
* Данные по кормам.
* Исторические показатели.
* Демо-данные (используются для демонстрации работы прототипа).

## 🚧 Ограничения MVP (хакатонной версии)

Мы честно оцениваем текущую стадию разработки:
- Часть сигналов в модели пока является proxy-оценкой, а не полноценной реальной метрикой.
- ML-компонент помогает в ранжировании, но не является единственным источником истины.
- Некоторые эко/гео показатели считаются на основе приближённых данных.
- **Система не заменяет комиссию, а помогает ей принимать решение.**
